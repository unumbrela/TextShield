### 1. 已有贡献回顾
| 贡献 | 内容 |
|------|------|
| Forensic Continual Pre-training | 利用自然图像伪造数据集进行由易到难的课程学习预训练 |
| GRPO 强化学习微调 | 5个奖励函数（分类/方法/定位/OCR/格式），减少标注依赖 |
| OCR Rectification | 推理时利用OCR引擎修正定位预测 |
| TFR Benchmark | 45k+图像，16种语言，10种篡改方法的综合评测基准 |

### 2. 现有局限性分析

| 编号 | 局限性 | 具体表现 | 改进空间 |
|------|--------|---------|---------|
| L1 | **单轮推理，无自校验** | 模型一次性输出结果，无法检查和修正自身错误 | 引入自反思/验证循环 |
| L2 | **定位精度依赖外部OCR** | OCR Rectification 是后处理方案，与模型解耦 | 端到端定位增强 |
| L3 | **奖励函数粗粒度** | 仅有结果级(outcome-level)奖励，缺少推理过程监督 | 过程级奖励(Process Reward) |
| L4 | **单尺度视觉感知** | 固定分辨率输入，对微小篡改痕迹感知不足 | 多尺度/局部放大策略 |
| L5 | **推理链质量不可控** | GRPO只约束最终答案，推理过程质量无保证 | 推理链质量奖励 |
| L6 | **跨域泛化不足** | CIS/CTM/CL集上性能下降明显 | 域自适应/对比学习 |
| L7 | **仅支持Bounding Box** | 输出矩形框，无法精确描述不规则篡改区域 | 像素级/多边形定位 |

## 二、创新点设计

### 方案 A：Self-Reflective Forensic Reasoning（自反思取证推理）⭐ 推荐主线

**核心思想**：让MLLM具备"推理-验证-修正"的多轮自反思能力，模拟人类专家的取证思维过程。

**创新点清单**：

#### 创新点 1：Reflective Reasoning Framework（反思推理框架）
- **动机**：人类取证专家会反复检查自己的判断，但当前MLLM是单次推理
- **方法**：设计三阶段推理框架：
  1. **Initial Analysis（初步分析）**：`<think>` 初步推理 `</think>` → `<answer>` 初步答案 `</answer>`
  2. **Self-Verification（自我验证）**：`<verify>` 对初步答案进行质疑和检查 `</verify>`
  3. **Refined Answer（修正答案）**：`<refine>` 根据验证结果修正或确认 `</refine>` → `<final_answer>` `</final_answer>`
- **技术实现**：在GRPO中增加验证阶段的奖励函数，鼓励模型发现并修正自身错误

#### 创新点 2：Process Reward Model for Forensic Reasoning（取证推理过程奖励模型）
- **动机**：当前GRPO仅有outcome-level奖励，无法保证推理过程质量
- **方法**：
  - 训练一个轻量级 Process Reward Model (PRM)，对推理链的每一步评分
  - PRM评估维度：(a) 证据引用准确性 (b) 逻辑一致性 (c) 结论合理性
  - 将PRM分数作为GRPO的额外奖励信号
- **实现**：
  - 利用GPT-4o对推理链进行step-level标注（好/坏/中性），构建PRM训练数据
  - PRM可基于小型LLM (如Qwen2.5-1.5B) 微调

#### 创新点 3：Zoom-in Forensic Analysis（局部放大取证分析）
- **动机**：文字篡改痕迹往往很细微（边缘不自然、纹理不一致），全局分辨率不够
- **方法**：
  - 第一轮全局分析，识别可疑区域
  - 自动裁剪可疑区域并放大（2x-4x）
  - 第二轮对放大区域进行细粒度分析
  - 融合全局+局部信息得出最终判断
- **技术优势**：充分利用Qwen2.5-VL的多图输入能力，无需架构修改

#### 创新点 4：Consistency-Aware Reward（一致性感知奖励）
- **动机**：当前奖励函数相互独立，不考虑预测结果内部的逻辑一致性
- **方法**：新增一致性奖励函数：
  - 如果预测为"real"但输出了篡改位置 → 惩罚
  - 如果预测为"tampered"但推理过程中未提及任何视觉证据 → 惩罚
  - 推理过程中提到的篡改方法与最终答案不一致 → 惩罚
  - OCR内容与检测到的语言不匹配 → 惩罚

---

### 方案 B：Multi-Scale Forensic Perception（多尺度取证感知）备选主线

**核心思想**：设计多尺度感知策略，从不同粒度捕捉篡改痕迹。

| 创新点 | 内容 |
|--------|------|
| Coarse-to-Fine Detection | 先全局检测可疑区域，再局部精细化分析 |
| Scale-Aware Reward | 针对不同尺度预测设计专门的奖励函数 |
| Multi-Resolution Pre-training | 在预训练阶段引入多分辨率数据增强 |

---

### 方案 C：Collaborative Forensic Agents（协作取证智能体）备选主线

**核心思想**：将取证任务分解为多个专家子任务，由不同角色的Agent协作完成。

| 创新点 | 内容 |
|--------|------|
| Detective Agent | 负责整体真伪判断和篡改方法识别 |
| Locator Agent | 负责精确定位篡改区域 |
| Reader Agent | 负责识别篡改文字内容 |
| Arbitrator Agent | 综合各Agent意见做最终裁决 |

---

### 推荐方案：A为主线，融合A3（Zoom-in）作为辅助

**最终创新点凝练**：

1. **Self-Reflective Forensic Reasoning（核心贡献）**：首个在篡改文本检测中引入自反思推理的方法，通过"分析-验证-修正"三阶段框架提升推理质量和准确性
2. **Process Reward Model for Forensic Reasoning（核心贡献）**：首个针对取证推理链的过程级奖励模型，从证据准确性、逻辑一致性、结论合理性三个维度评估推理质量
3. **Zoom-in Forensic Analysis（辅助贡献）**：利用MLLM多图能力实现局部放大分析，提升微细篡改痕迹的检测能力
4. **Consistency-Aware Reward（辅助贡献）**：约束预测结果内部的逻辑一致性，减少自相矛盾的输出

**一致性奖励函数设计**：

```python
def consistency_reward(prediction):
    """
    检查预测结果的内部逻辑一致性
    """
    score = 1.0
    cls = extract_classification(prediction)
    has_bbox = has_bounding_box(prediction)
    has_ocr = has_ocr_content(prediction)
    has_method = has_tampering_method(prediction)

    # 规则1: 判断为real但输出了篡改信息
    if cls == "real" and (has_bbox or has_ocr or has_method):
        score -= 0.5

    # 规则2: 判断为tampered但缺少必要信息
    if cls == "tampered" and not (has_bbox and has_ocr and has_method):
        score -= 0.3

    # 规则3: think和answer中的分类不一致
    think_cls = extract_think_classification(prediction)
    if think_cls and think_cls != cls:
        score -= 0.3

    return max(0, score)
```

**反思质量奖励设计**：

```python
def reflection_reward(prediction, ground_truth):
    """
    奖励模型在verify阶段发现并修正错误
    """
    initial_answer = extract_initial_answer(prediction)
    final_answer = extract_final_answer(prediction)
    verify_content = extract_verify(prediction)

    if not verify_content:
        return 0.0  # 没有验证过程，不奖励

    initial_correct = is_correct(initial_answer, ground_truth)
    final_correct = is_correct(final_answer, ground_truth)

    if not initial_correct and final_correct:
        return 1.0  # 成功修正错误，最高奖励
    elif initial_correct and final_correct:
        return 0.7  # 正确确认，较高奖励
    elif initial_correct and not final_correct:
        return 0.0  # 修正反而错了，不奖励
    else:
        return 0.2  # 都错但尝试了验证
```

### 故事线与创新点定位

**核心叙事**：TextShield-R1 虽然首次将强化学习引入篡改文本检测，但其单轮推理模式和粗粒度奖励限制了推理质量和检测精度。人类取证专家总是会反复检查自己的判断——我们将这种自反思能力赋予MLLM，并通过过程级奖励模型确保推理链质量。

**贡献列表**：
1. We propose a Self-Reflective Forensic Reasoning framework that enables MLLMs to analyze, verify, and refine their predictions through a structured three-stage reasoning process, significantly improving detection accuracy and reasoning quality.
2. We introduce a Process Reward Model specifically designed for forensic reasoning, which evaluates reasoning chains at the step level across evidence accuracy, logical consistency, and conclusion soundness, providing fine-grained supervision for GRPO training.
3. We design a Zoom-in Forensic Analysis strategy that leverages MLLMs' multi-image capabilities to perform coarse-to-fine detection, enhancing the perception of subtle tampering artifacts.
4. We propose Consistency-Aware Rewards that enforce logical coherence among classification, localization, OCR, and reasoning outputs, reducing self-contradictory predictions.

### 新增 Format Reward（更新版）

```python
def format_reward_v2(prediction):
    """扩展格式奖励，包含反思格式"""
    has_think = "<think>" in prediction and "</think>" in prediction
    has_answer = "<answer>" in prediction and "</answer>" in prediction
    has_verify = "<verify>" in prediction and "</verify>" in prediction
    has_final = "<final_answer>" in prediction and "</final_answer>" in prediction

    if has_think and has_answer and has_verify and has_final:
        return 1.0  # 完整反思格式
    elif has_think and has_answer:
        return 0.5  # 基础格式（向后兼容）
    else:
        return 0.0
```

## 十、风险评估与预案

| 风险 | 影响 | 预案 |
|------|------|------|
| 反思机制导致推理延迟过大 | 实用性受质疑 | 分析延迟开销，设计Early Exit策略（高置信度时跳过verify） |
| PRM训练数据质量不足 | PRM效果不明显 | 退回使用规则化的推理质量评估指标替代PRM |
| Zoom-in效果不显著 | 创新点减弱 | 作为辅助贡献，主打反思+PRM两个核心创新点 |
| GRPO训练不稳定 | 性能波动 | 多次重复实验取平均，调整奖励权重 |
| 反思格式导致GRPO收敛困难 | 训练失败 | 增加反思SFT冷启动的数据量，降低反思奖励权重 |
| 改进幅度不够显著 | 论文说服力不足 | 聚焦推理质量提升（Res.指标）而非分类准确率，展示定性分析 |
